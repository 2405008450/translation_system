# 云服务器部署常用指令

项目目录示例：`~/opt/translation_system`

## 首次部署

```bash
cd ~/opt/translation_system
git pull
cp .env.prod.example .env.prod
nano .env.prod   # 改密码、JWT、LLM Key、CORS 公网 IP

chmod +x scripts/deploy_prod.sh

# 标准（方案 A：不启 nginx，直连 19013）
scripts/deploy_prod.sh up

# 需要 Mihomo 代理时
USE_PROXY=1 scripts/deploy_prod.sh up
```

安全组放行 **TCP 19013**。

## 代码更新 / 重新构建

```bash
cd ~/opt/translation_system
git pull

echo "=== 当前版本 ==="
git log -1 --oneline

# 重建 app 镜像并滚动 app + worker
scripts/deploy_prod.sh restart

# 查看状态与日志
scripts/deploy_prod.sh ps
scripts/deploy_prod.sh logs
scripts/deploy_prod.sh health
```

手动等价命令：

```bash
sudo docker-compose --env-file .env.prod -f docker-compose.prod.yml -f docker-compose.proxy.yml build --no-cache app
sudo docker-compose --env-file .env.prod -f docker-compose.prod.yml -f docker-compose.proxy.yml up -d --force-recreate --no-deps app worker
```

## 验证上传与存储

```bash
# 健康检查（方案 A）
curl -fsS http://127.0.0.1:19013/api/health

# 上传能力（含 limits）
curl -fsS http://127.0.0.1/api/file-records/upload-capabilities | head -c 500

# 导入暂存目录（共享卷）
sudo docker exec ai-translation-app ls -la /app/data/import_tasks
sudo docker exec ai-translation-worker ls -la /app/data/import_tasks
```

## Mihomo 代理排查

```bash
sudo docker-compose --env-file .env.prod -f docker-compose.prod.yml -f docker-compose.proxy.yml ps mihomo

sudo docker-compose --env-file .env.prod -f docker-compose.prod.yml -f docker-compose.proxy.yml exec -T app \
  sh -lc 'env | grep -i proxy'
```
