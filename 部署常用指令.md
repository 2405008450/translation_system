# 云服务器部署常用指令

项目目录示例：`~/opt/translation_system`

## 首次部署

```bash
cd ~/opt/translation_system
git pull
cp .env.prod.example .env.prod
nano .env.prod   # 改密码、JWT、LLM Key、CORS 公网 IP

chmod +x scripts/deploy_prod.sh

# 标准：默认启 nginx，对外发布 80
scripts/deploy_prod.sh up

# 需要 Mihomo 代理时
USE_PROXY=1 scripts/deploy_prod.sh up
```

安全组放行 **TCP 80**。

## 代码更新 / 重新构建

```bash
cd ~/opt/translation_system
git pull

echo "=== 当前版本 ==="
git log -1 --oneline

# 重建 app 镜像并滚动 app + 各类 worker
scripts/deploy_prod.sh restart

# 查看状态与日志
scripts/deploy_prod.sh ps
scripts/deploy_prod.sh logs
scripts/deploy_prod.sh health
```

手动等价命令：

```bash
sudo docker-compose --env-file .env.prod -f docker-compose.prod.yml -f docker-compose.proxy.yml -f docker-compose.nginx.yml build --no-cache app
sudo docker-compose --env-file .env.prod -f docker-compose.prod.yml -f docker-compose.proxy.yml -f docker-compose.nginx.yml up --force-recreate db-migrate
sudo docker-compose --env-file .env.prod -f docker-compose.prod.yml -f docker-compose.proxy.yml -f docker-compose.nginx.yml up -d --force-recreate --no-deps app import-worker worker auto-tm-worker segment-sync-worker pretranslation-worker nginx
```

## 验证上传与存储

```bash
# 健康检查
curl -fsS http://127.0.0.1/api/health

# 上传能力（含 limits）
curl -fsS http://127.0.0.1/api/file-records/upload-capabilities | head -c 500

# 导入暂存目录（共享卷）
sudo docker exec ai-translation-app ls -la /app/data/import_tasks
sudo docker exec ai-translation-import-worker ls -la /app/data/import_tasks
sudo docker exec ai-translation-worker ls -la /app/data/import_tasks
sudo docker exec ai-translation-auto-tm-worker ls -la /app/data/import_tasks
sudo docker exec ai-translation-segment-sync-worker ls -la /app/data/import_tasks

# 队列长度：上传文件、记忆库、术语库、词汇表导入走 arq:import
sudo docker exec ai-translation-redis redis-cli ZCARD arq:import
sudo docker exec ai-translation-redis redis-cli ZCARD arq:maintenance
sudo docker exec ai-translation-redis redis-cli ZCARD arq:auto-tm
sudo docker exec ai-translation-redis redis-cli ZCARD arq:segment-sync
sudo docker exec ai-translation-redis redis-cli ZCARD arq:pretranslation
```

## 并发参数第一档

```env
ARQ_IMPORT_MAX_JOBS=2
ARQ_MAINTENANCE_MAX_JOBS=2
ARQ_AUTO_TM_MAX_JOBS=1
ARQ_SEGMENT_SYNC_MAX_JOBS=1
ARQ_PRETRANSLATION_MAX_JOBS=2
PRETRANSLATION_RUN_FILE_CONCURRENCY=2
AUTO_TM_OUTBOX_MAX_BATCHES_PER_RUN=5
AUTO_TM_REMATCH_MAX_FILES_PER_RUN=1
LLM_MAX_CONCURRENCY=3
```

预翻译理论请求上限约为 `ARQ_PRETRANSLATION_MAX_JOBS × PRETRANSLATION_RUN_FILE_CONCURRENCY × LLM_MAX_CONCURRENCY`。如果模型接口出现 429、超时或错误率升高，先把 `LLM_MAX_CONCURRENCY` 降回 2；如果服务器仍很轻松且模型接口稳定，再逐步提高。
