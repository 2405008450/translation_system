# 项目基本框架

> 项目名称：**AI Translation System**（项目化翻译工作台 + TM/术语资产管理）
>
> 技术栈：**FastAPI + SQLAlchemy 2 + PostgreSQL(pg_trgm/pgvector) + Vue 3 + Vite + Pinia + vue-i18n**
>
> 定位：一个面向内部翻译流程的 Web 工作台，支持项目创建、源文档上传、多格式句段抽取、TM/术语辅助、AI 修正、人工审校、批注协作、修订确认与译后文件导出。

## 3. 后端模块职责

### 3.1 入口、配置与运行时检查

| 文件 | 作用 |
| --- | --- |
| `app/main.py` | 配置日志；加载并校验 `Settings`；调用 `ensure_runtime_schema()`；挂载 `auth`、主业务、术语路由；生产环境托管 `frontend/dist` |
| `app/config.py` | 管理 `DATABASE_URL`、`FILE_STORAGE_DIR`、上传大小、TM 阈值、Redis、TM 向量、DeepSeek/OpenRouter、JWT、CORS、日志级别；若 `JWT_SECRET_KEY` 仍是默认值会拒绝启动 |
| `app/database.py` | SQLAlchemy 2 engine、`SessionLocal`、`Base` 与 `get_db()` |
| `app/logging.py` | 后端日志格式与级别 |
| `app/services/schema_setup.py` | 启动时检查新表名、关键表和列；可补齐部分新 schema，旧 TM 表名会提示先执行重命名脚本 |

### 3.2 数据模型（`app/models.py`）

| 表 / 模型 | 说明 |
| --- | --- |
| `users` / `User` | 用户账号，含 `username`、`nickname`、`role`、`is_active` |
| `projects` / `Project` | 翻译项目，含语言对、截止时间、访问级别、创建人、翻译细则 `translation_guidelines` |
| `file_records` / `FileRecord` | 项目下的任务文件，保存文件名、哈希、状态、解析模式、语言对、绑定 TM/术语库 |
| `segments` / `Segment` | 任务句段，保存源文、显示文本、译文、状态、分数、匹配来源、文档结构坐标 |
| `segment_revisions` / `SegmentRevision` | 每次人工或 LLM 写回生成的修改快照，支持 pending / accepted / rejected |
| `segment_comments` / `SegmentComment` | 批注与回复，支持句段锚点或选区锚点、open / resolved 状态 |
| `memory_bases` / `TMCollection` | TM 记忆库分组，按语言对管理 |
| `memory_entries` / `TranslationMemory` | TM 条目，含 `source_hash`、`source_normalized`、语言对、创建者、pg_trgm 索引与 pgvector embedding |
| `term_bases` / `TermBase` | 术语库，按语言对管理 |
| `term_entries` / `TermEntry` | 术语条目，含归一化源术语和创建者 |

> 旧命名 `tm_collections` / `translation_memory` 已迁移为 `memory_bases` / `memory_entries`；代码中保留 `MemoryBase` / `MemoryEntry` 别名便于过渡。

### 3.3 认证与权限

- `app/auth.py` 使用 `passlib[bcrypt]` 哈希密码，使用 `python-jose` 生成和校验 JWT。
- `get_current_user` 保护业务路由，`require_admin` 用于管理员能力。
- `app/routers/auth.py`：

| 方法 | 路径 | 说明 |
| --- | --- | --- |
| `GET` | `/api/auth/init` | 查询是否需要首次初始化管理员 |
| `POST` | `/api/auth/init` | 创建首个管理员并返回 JWT |
| `POST` | `/api/auth/login` | 登录 |
| `GET` | `/api/auth/me` | 获取当前用户 |
| `GET` | `/api/auth/users` | 用户列表 |
| `POST` | `/api/auth/register` | 管理员创建用户 |
| `PATCH` | `/api/auth/users/{id}` | 更新用户名、昵称或密码 |

### 3.4 主业务 API（`app/routers/api.py`）

所有业务接口默认挂在 `/api` 下并要求登录。新接口以 `file-records`、`projects`、`translation-memory` 为主；`documents` 和 `/tm/*` 保留隐藏兼容别名。

| 分组 | 主要路由 |
| --- | --- |
| 解析 | `POST /api/parser/slate`、`POST /api/parser/workspace`、`POST /api/parser/parse`、`GET /api/parser/formats` |
| 适配器导出 | `POST /api/export/txt`、`POST /api/export/docx`、`POST /api/export/dita`、`POST /api/export/svg`、`POST /api/export/tmx`、`POST /api/export/xliff`、`POST /api/import/xliff`、`GET /api/export/formats` |
| 项目 | `GET/POST /api/projects`、`GET/PATCH/DELETE /api/projects/{id}`、`POST /api/projects/{id}/source-document` |
| 任务文件 | `POST /api/file-records`、`GET /api/file-records`、`GET /api/file-records/{id}`、`DELETE /api/file-records/{id}` |
| 预览与导出 | `GET /api/file-records/{id}/preview`、`GET /api/file-records/{id}/export`、`GET /api/file-records/{id}/export-options`、`GET /api/file-records/{id}/export/{type}` |
| 句段 | `PUT /api/file-records/{id}/segments/{sentence_id}`、`PUT /api/file-records/{id}/segments` |
| 匹配与绑定 | `GET /api/file-records/{id}/segments/{segment_ref}/tm-candidates`、`POST /api/file-records/{id}/rematch`、`PATCH /api/file-records/{id}/bindings` |
| 保存到 TM | `POST /api/file-records/{id}/save-to-tm` |
| 修订 | `GET /api/file-records/{id}/revisions`、`PATCH /api/revisions/{revision_id}`、`POST /api/file-records/{id}/revisions/batch-accept`、`POST /api/file-records/{id}/revisions/batch-reject` |
| 批注 | `GET/POST /api/file-records/{id}/comments`、`PATCH/DELETE /api/comments/{id}`、`POST /api/comments/{id}/replies` |
| AI 修正 | `POST /api/file-records/{id}/llm-translate`，SSE 事件：`start` / `segment` / `error` / `complete` |
| LLM 细则模板 | `GET /api/guideline-templates`、`GET /api/guideline-templates/{id}`、`POST /api/guideline-templates/import`；导入文件保存到 `prompt_templates/translation_guidelines/`，手动输入只作为本次临时提示词 |
| TM | `GET/POST /api/translation-memory/collections`、`GET/PUT/DELETE /api/translation-memory/collections/{id}`、`POST /api/translation-memory/collections/merge`、`GET/POST /api/translation-memory/collections/{id}/entries`、`PUT/DELETE /api/translation-memory/entries/{id}`、`POST /api/translation-memory/import-xlsx`、`POST /api/translation-memory/preview-sdltm`、`GET /api/translation-memory/collections/{id}/export-xlsx` |
| 旧术语接口 | `/api/termbase/*` 保留给工作台术语匹配等旧路径 |

### 3.5 术语库 API（`app/routers/term_base.py`）

| 方法 | 路径 | 说明 |
| --- | --- | --- |
| `GET/POST` | `/api/term-bases` | 术语库列表 / 新建 |
| `POST` | `/api/term-bases/merge` | 合并多个语言对一致的术语库 |

### 3.6 服务层

| 模块 | 职责 |
| --- | --- |
| `task_file_service.py` | 判断任务格式；DOCX 走专用流程，其它文件走 adapters；统一生成工作台数据、预览 HTML 与译后文件 |
| `document_workspace.py` | DOCX 深度解析，覆盖正文、表格、页眉页脚、脚注尾注、文本框、编号、分页、字段、OMML 数学公式占位符与 MathML 预览；支持 `full` / `body_only` |
| `document_exporter.py` | 将句段译文回写 DOCX，保留未翻译内容与部分原始格式 |
| `document_storage.py` | 源文件存储在 `data/file_records/`，按任务 ID 管理 |
| `file_record_service.py` | 创建任务与句段、分页加载、状态/进度计算、批量更新、LLM 写回、删除任务 |
| `matcher.py` | 批量 TM 匹配：`source_hash` 精确匹配、`pg_trgm` 模糊召回、`SequenceMatcher` 重排、pgvector 语义候选融合，返回 Top 候选与 diff |
| `tm_vector.py` | 本地确定性 embedding、向量维度配置、embedding SQL 字面量、可用性标记与同步 |
| `tm_importer.py` | XLSX / SDLTM 导入 TM，跳过表头和空行，写入语言对、创建者、归一化字段，并同步 embedding |
| `term_importer.py` | XLSX 导入术语库条目，写入语言对和归一化字段 |
| `revision_service.py` | 生成修改快照，接受/拒绝单条或批量修订 |
| `comment_service.py` | 批注列表、新建、编辑、删除、回复、序列化用户信息 |
| `guideline_repository.py` | 管理仓库内可复用翻译细则模板；导入 `.md` / `.txt` 文本后统一转存为 UTF-8 Markdown，供后续 LLM 请求复用 |
| `llm_service.py` | Provider 选择、`auto` fallback、并发限制、重试、单条/批量提示词、可复用 Markdown 细则与本次临时提示词注入、数字/符号/数学占位符校验 |
| `cache.py` | Redis 可选缓存，未配置时回退进程内存 |
| `language_pairs.py` | 语言别名规范化、语言对一致性校验、显示标签 |
| `xlsx_exporter.py` | 生成 TM / 术语 XLSX 下载响应 |

### 3.7 多格式适配器（`app/services/adapters/`）

适配器系统围绕 `FormatAdapter`、`AdapterRegistry`、`DocumentAST`、`Segment`、`ParseResult` 工作：

- `registry.py` 管理扩展名到适配器实例的映射。
- `__init__.py` 注册默认适配器：txt、docx、xlsx、pdf、pptx、dita、svg、yaml、json、php、html、properties、po、strings、markdown、srt、csv、sdlxliff、txml、dxf、zip、idml、mif、rar 等。
- `task_file_service.py` 当前任务入口白名单只开放：

```text
.docx
.txt, .csv
.html, .htm, .md, .markdown
.json, .yaml, .yml, .php, .properties
.po, .pot, .strings, .srt
.dita, .ditamap, .xml, .svg
.sdlxliff, .txml
.dxf, .idml, .mif
.zip
```

> 代码中有 PDF / PPTX / XLSX / RAR 等适配器文件，但当前任务上传白名单和前端选择尚未开放这些格式。

## 4. 前端模块职责

### 4.1 技术栈

- Vue 3 + `<script setup>` + TypeScript
- Vue Router 4：history 模式、登录守卫
- Pinia：登录态、任务、工作台句段、批注、外壳状态、偏好设置
- Axios：`src/api/http.ts` 统一 `baseURL=/api`，请求拦截注入 JWT，响应拦截显示错误 toast
- vue-i18n：中文文案集中在 `src/locales/zh-CN.ts`
- lucide-vue-next：按钮和导航图标
- Vite：开发代理与生产构建

## 9. 关键约定

1. **接口命名**：新开发优先使用 `/api/projects`、`/api/file-records`、`/api/translation-memory`、`/api/term-bases`；`/api/documents`、`/api/tm`、部分 `/api/termbase` 属于兼容路径。
2. **语言对**：项目、任务、TM 记忆库、术语库都要保持源语言/目标语言一致；后端通过 `language_pairs.py` 规范化别名。
3. **解析模式**：DOCX 支持 `full` 和 `body_only`；`full` 包含页眉页脚、脚注尾注、文本框等扩展 story。
4. **任务文件白名单**：前后端当前开放的任务格式以 `constants/taskFiles.ts` 和 `task_file_service.TASK_ADAPTER_EXTENSIONS` 为准。
5. **句段状态**：常见状态包括 `exact`、`fuzzy`、`none`、`confirmed`；`source` 字段记录 `tm`、`manual`、`llm` 等来源。
6. **修订机制**：人工编辑和 LLM 写回都会创建 `segment_revisions`，前端可接受或拒绝。
7. **批注机制**：批注支持句段级和选区级锚点，回复通过 `parent_id` 形成线程。
8. **TM 匹配策略**：精确匹配只认 `source_hash`；模糊匹配混合 trigram、字符串相似度和可选 pgvector 得分。
9. **AI 输出保护**：LLM 结果会校验数字片段、复选框/特殊符号和数学公式占位符，失败会通过 SSE `error` 返回。
10. **LLM 细则来源**：可复用细则使用 Markdown，导入 `.md` / `.txt` 后统一以 UTF-8 `.md` 保存到 `prompt_templates/translation_guidelines/`；用户在工作台或预翻译中手动输入的内容只作为本次临时提示词，不写入模板仓库。
11. **安全配置**：`.env` 不入库；`JWT_SECRET_KEY` 必须替换；LLM API Key 至少配置一个才可使用 AI 修正。
12. **中文编码**：代码和文档按 UTF-8 处理，中文注释、错误提示和前端文案统一避免乱码。

## 10. 大文档分页与性能策略

任务落地页 `/tasks/:id` 已按“大文档优先”调整：浏览器端不再依赖一次性加载全部句段，避免几十万字任务在个人电脑上因前端内存和 DOM 规模过大而卡死。

### 10.1 工作台分页

- `GET /api/file-records/{id}` 保留任务元信息和兼容性初始页。
- `GET /api/file-records/{id}/segments` 是工作台句段分页主接口，支持 `skip`、`limit`、`scope`、`source_query`、`target_query`、`search_fuzzy`。
- 前端 `segment.ts` 只保留当前页句段、当前筛选条件、分页信息和待同步修改；默认页大小为 200，可切换 100 / 200 / 500 / 1000。
- `WorkbenchView.vue` 使用分页控件 + `VirtualList.vue`，当前页内继续虚拟滚动，但不再滚动累积全文。

### 10.2 窗口预览与全文操作

- `GET /api/file-records/{id}/preview` 支持 `skip`、`limit`、`mode=source|target`，返回当前页窗口 HTML；大文档不再默认返回完整预览 HTML。
- 搜索和筛选在服务端分页执行，前端只展示当前结果页。
- “全部替换”走 `POST /api/file-records/{id}/segments/replace`，由后端按筛选条件处理全文。
- “保存到 TM”走后端任务级接口，前端通过 `GET /api/file-records/{id}/save-to-tm/stats` 获取统计，不再为了统计加载全文。
- 修订和批注接口支持按当前页 `skip` / `limit` / 筛选条件查询，由后端定位窗口内句段，避免前端把数百到上千个 `sentence_id` 拼进 URL。

### 10.3 Redis / 队列 / 数据库索引

- `REDIS_URL` 建议在生产环境必配；否则 DOCX 解析缓存会回退到 FastAPI 进程内存，大文档会抬高后端内存水位。
- `IMPORT_QUEUE_BACKEND=arq` 可把导入任务投递到 Redis + ARQ worker，避免大文件导入长时间占用 Web 请求进程。
- `segments` 增加 `ix_segments_file_record_order(file_record_id, block_index, row_index, cell_index, sentence_id)`，用于任务页稳定分页排序。
- 已有库可执行 `scripts/add_segments_pagination_index.sql` 补齐分页索引。
