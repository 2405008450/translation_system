# Frontend UI Style Guide

本文件用于约束后续前端开发的界面风格，目标是让新页面和现有登录页、左侧导航应用壳、任务页、工作台保持同一套视觉语言。

## 1. 设计基调

- 产品类型：工具型工作台，不做营销页风格。
- 整体气质：克制、清晰、偏专业，不做炫技动画和重装饰。
- 视觉关键词：浅色背景、低饱和中性色、绿色主操作色、轻边框、稳定布局。

## 2. 颜色规范

以当前实现为准，后续新增页面优先复用下面的颜色：

- 主文字：`#17313b`
- 次级文字：`#48646d` / `#5a7074` / `#607677`
- 主按钮 / 重点色：`#0d7a68`
- 主按钮文字：`#ffffff`
- 页面背景渐变：
  - `#f5f8f3`
  - `#eef3ef`
  - `#edf1f6`
- 面板背景：`rgba(255, 255, 255, 0.94)`
- 常规边框：`#ced8d5`
- 输入框边框：`#b8c9cc`
- 列表项浅底：`#fbfdfc`
- 错误色：`#b93030`
- 预览高亮：`#f3dd86`

要求：

- 继续使用绿色作为唯一主操作色。
- 不引入紫色主视觉、深蓝整页主题、米黄色整页主题。
- 除状态色外，不新增高饱和强调色。

## 3. 排版规范

- 字体栈：`"Segoe UI", "PingFang SC", "Microsoft YaHei", sans-serif`
- 页面主标题：`30px`
- 区块标题：`16px`, `font-weight: 600`
- 正文：默认 `14px - 16px`
- 辅助说明：`13px`
- 英文全部大写仅用于小型 kicker，例如 `.section-kicker`
- 不使用负字距，不使用随视口缩放的字体

## 4. 圆角、边框、阴影

- 统一圆角：`6px` 或 `8px`
- 输入框 / 按钮：`6px`
- 面板 / 列表项 / 统计块：`8px`
- 边框优先于重阴影
- 阴影只用于登录面板这类独立浮层，且保持轻量

## 5. 页面结构

### 5.1 顶层结构

- 未登录页使用 `.page-shell` 或独立居中布局
- 已登录区统一使用“左侧导航 + 右侧内容区”应用壳
- 左侧导航负责板块切换，不在内容区重复做一级导航
- 内容区顶部使用统一的 section header，显示当前板块标题和说明
- 页面主体优先使用网格布局，不使用大面积悬浮卡片堆叠

### 5.2 导航规范

- 左侧导航使用固定宽度窄栏，桌面端保持常驻
- 导航项是轻量按钮块，不做大图标宫格
- 当前激活项通过边框和浅绿色背景表达
- 导航文字两层：
  - 第一层：板块名
  - 第二层：简短说明
- 移动端允许导航折叠为顶部块状导航，但语义仍然保持同一层级

### 5.3 面板规范

- 功能区使用 `.panel`
- 工具栏使用 `.toolbar-panel`
- 面板内按“标题 -> 表单/内容 -> 结果反馈”的顺序组织

### 5.4 列表规范

- 任务列表使用纵向 `grid`
- 每个任务项是紧凑的信息块，内容在左，操作在右
- 列表项状态通过文字和轻标签表达，不靠大块背景色

## 6. 表单与操作

- 表单字段统一使用：
  - `.field`
  - `.field__label`
  - `.field__control`
- 主要动作使用 `.button--primary`
- 次要动作使用 `.button`
- 一个区域内只保留一个主按钮，其余都降级为次要按钮
- 错误和反馈统一走 `.form-message`

## 7. 工作台规范

- 编辑区和预览区并列，不做弹窗式主流程
- 长列表必须优先保证滚动性能，继续使用虚拟列表
- 句段块保持固定高度，避免输入时抖动
- 当前聚焦句段要有边框或背景高亮，同时联动预览区
- 自动保存状态、AI 处理状态放在工具栏，不打断主编辑流程

## 8. 文案风格

- 直接描述动作和结果，不写营销话术
- 优先使用短句：
  - “上传并进入工作台”
  - “导入记忆库”
  - “任务列表加载中...”
- 错误提示明确说明原因，避免“发生未知错误”这类空话

## 9. 新页面开发规则

新增页面时优先复用现有类名和结构：

- 页面壳：`.page-shell`
- 应用壳：`.app-frame` / `.app-sidebar` / `.app-main`
- 面板：`.panel`
- 表单：`.field` / `.field__control`
- 按钮：`.button` / `.button--primary`
- 反馈：`.form-message`

如果需要新增样式，遵守下面几点：

- 先扩展现有类，再新增新类
- 新类命名保持语义化，例如 `xxx-grid`、`xxx-item`、`xxx-actions`
- 先补布局类，再补颜色和装饰
- 新增颜色前先检查能否复用现有 token

## 10. 明确禁止

- 不做渐变按钮、发光按钮、玻璃拟态面板
- 不做紫色或深蓝主色重主题
- 不做超过 `8px` 的大圆角
- 不做卡片套卡片的层层包裹
- 不把主工作区塞进装饰性容器里
- 不新增无实际用途的插画、徽章、空装饰块

## 11. 开发自检清单

提交前至少检查以下内容：

- 新页面是否复用了现有面板、按钮、表单风格
- 主按钮是否仍然只有一个视觉主操作
- 手机宽度下是否没有文字溢出和布局跳动
- 是否引入了不属于当前体系的新主色
- 列表、表单、编辑区是否保持稳定尺寸
- 文案是否仍然是工具型产品文案
